# 产品公告

本章节介绍了阿里云通义点金（Qwen-Dianjin）重要产品公告，敬请关注。

尊敬的用户，

感谢您一直以来对通义点金产品的支持和信任。随着大模型应用场景不断深化，为更好地满足客户在数据管理、模型迭代和智能服务等方面的需求，我们持续对点金产品进行功能演进与能力升级。本次更新将新增“数据飞轮”相关服务能力，并对现有RAG（检索增强生成）等功能进行优化。

为保障服务资源的合理使用与公平计费，我们在保留原有 Agent Tokens 资源消耗量 计费项的基础上，将新增以下两项计费维度：

1.  数据飞轮服务资源消耗量  
    客户在使用数据飞轮进行模型后训练（Post-training）时，将按实际使用的计算实例资源计费。
    
    -   计量单位：实例·小时
        
    -   计费规则：不足1小时按1小时计。
        
2.  存储资源占用量  
    在客户使用 RAG 功能或资源库时，上传文档、生成向量索引或进行搜索缓存等操作，将产生存储资源占用。
    
    -   计量单位：GB·小时
        
    -   计费规则：不足1小时按1小时计。
        

-   示例1：数据飞轮训练  
    用户小A使用数据飞轮进行模型后训练，调用了2个实例，训练耗时3.5小时。  
    → 数据飞轮服务资源消耗量 = 2 实例 × 4 小时 = 8 实例·小时  
    （注：3.5小时向上取整为4小时）
    
-   示例2：RAG 存储占用  
    用户小A上传了2GB原始文档，并通过RAG功能完成向量化处理，系统实际占用存储空间2.5GB，持续1小时。  
    → 存储资源占用量 = 2.5 GB × 1 小时 = 2.5 GB·小时
    
    （注：在一个小时的计费周期内，如果存储空间波动，则以计费期间某个统计时间点结果为准）
    

上线时间安排如下：

数据飞轮服务资源消耗量上线日期：2025年11月13日

功能影响：无

存储资源占用量上线日期：2026年1月01日00时

功能影响：对于目前已使用通义点金的存量客户会产生额外的存储资源占用量费用。

建议：为了最大限度地减少对您的影响，我们建议您在12月30日之前完成当前的打包任务，提前做好相应的调整和准备。

我们理解这一调整可能会给您带来不便，对此敬请谅解。
