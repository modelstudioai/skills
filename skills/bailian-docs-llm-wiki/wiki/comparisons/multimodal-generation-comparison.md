# 图像、视频与3D生成对比

## 背景与目的
百炼平台已构建覆盖静态视觉、动态视频与三维空间内容的全栈 AIGC 生成能力。随着业务场景从图文排版、短视频创作向虚拟化身、数字资产与实时渲染延伸，开发者在技术选型时常面临调用范式、延迟预期、资源持久化策略及成本模型的差异。本文档旨在横向对比**图像生成**、**视频生成**与**3D生成**三大模块的核心技术维度，为架构师与后端/客户端开发者提供清晰的集成边界与选型依据，帮助团队快速落地符合业务 SLA 的生成管线。

## 核心维度对比

| 对比维度 | [[图像生成]] | [[视频生成]] | [[3D生成]] |
|:---|:---|:---|:---|
| **输入格式** | 文本提示词、单图/多图公网 URL、Base64。新协议统一采用 `input.messages` 数组结构；旧版使用 `input.[[prompt|prompt]]`/`images` | 文本提示词、首帧/尾帧图像、参考图像、参考视频或音频。统一通过 `input.media` 数组传入，需声明 `type`（如 `first_frame`, `driving_audio`） | 文本（≤1024字符）、单张公网图 URL、4视角多图数组（固定顺序：前、左、后、右，缺失视角需填 `{}`） |
| **输出产物** | 图像 URL 或 Base64 编码。支持 1K~4K 分辨率，单次可请求 1~6 张，支持图文混排输出 | 视频 URL（MP4 等）。常见时长 5~15s，分辨率覆盖 480P~4K/1080P，支持多镜头续写与口型/动作对齐 | GLB 标准 3D 模型文件 + `.webp` 渲染预览图。面数最高支持 200 万（高精度模式），支持 PBR 材质与轻量化无贴图模型 |
| **支持模型/系列** | `qwen-image-2.0`（复杂文本/细节）、`wan2.6/2.7`（高清/混排）、`z-image-turbo`（极速）、`kling`（分镜组图）、`qwen-image-edit` 及垂类（试衣/写真/海报等） | `happyhorse`、`wan2.7-t2v/i2v`、`vidu`、`kling-v3`、`pixverse`，以及数字人（EMO/LivePortrait）与动作迁移专属模型 | `Tripo-H3.1`（高精度/高面数/150~200万面）、`Tripo-P1.0`（专业快速/≤2万面） |
| **API 端点与调用模式** | **支持同步/异步**。<br>同步：`/api/v1/services/aigc/multimodal-generation/generation`<br>异步：需 Header `X-DashScope-Async: enable`，标准任务轮询 | **仅支持异步**。<br>端点：`/api/v1/services/aigc/video-generation/video-synthesis`<br>强制携带 `X-DashScope-Async: enable` | **仅支持异步**。<br>端点：底层路由含 `/video-generation/`（历史命名，实为3D服务）<br>强制携带 `X-DashScope-Async: enable` |
| **计费方式** | 按**成功生成的输出图片张数**计费；主/子账号共享免费额度（默认500张）；QPS 通常限 2 | 按任务维度计费，与分辨率、时长、模型规格强绑定；异步任务 `task_id` 有效期 24h；高优场景需关注队列排队 | 按**成功任务数**计费（区分任务类型、面数精度、贴图质量）；下载链接有效期仅 **2小时**；仅支持中国内地（北京）地域 |
| **典型场景** | 电商主图/海报生成、复杂文本渲染、局部重绘/智能扩图、AI试衣与模特换装、分镜组图、带排版的图像翻译 | 社交媒体短视频创作、静态海报动态化、视频风格重绘、虚拟数字人播报、口型/肢体迁移、多模态指令编辑 | 游戏/元宇宙低面数资产快速生成、电商 3D 商品展示、VR/AR 原型验证、工业设计外观辅助建模、WebGL 轻量化加载 |
| **关键开发约束** | 新模型推荐同步直出；输入 URL 严禁含中文且需公网可达；强地域与 API Key 绑定；全量请求前置内容安全合规检测 | 分辨率与时长存在强耦合（如 480P 不支持 16:9）；垂类模型可能需前置合规检测（如 `-detect`）；耗时 1~5 分钟，需服务端异步队列管理 | 仅北京地域可用；4视角输入格式校验严格；模型下载链接必须业务侧及时持久化（OSS 等）；建议 15s 轮询间隔或改用回调机制 |

## 场景与技术选型建议

| 业务需求 | 推荐方案 | 集成与架构建议 |
|:---|:---|:---|
| **高实时交互、即时出图**（如编辑器实时预览、C端图文生成） | [[图像生成]] | 优先选用 `qwen-image-2.0` 或 `z-image-turbo` 的**同步接口**。客户端可直连或通过轻量网关转发，降低首字延迟（TTFT）。注意关闭默认水印需显式传参。 |
| **复杂商业物料、精细化修图与排版** | [[图像生成]] | 结合 `wan2.7` 的 `enable_interleave` 实现图文混排，或使用 `qwen-image-edit` 进行局部框选编辑。建议服务端实现重试逻辑，规避网络抖动导致的 `RateLimit`。 |
| **短视频自动化流水线、数字人交互** | [[视频生成]] | 采用**纯异步架构**：业务系统下发请求后记录 `task_id`，通过定时任务轮询或 Webhook 回调驱动状态流转。建议将 `[[prompt|prompt]]_extend` 关闭以降低排队时间，超长视频需依赖模型自带的续写能力拼接。 |
| **多视角参考、一致性角色生成** | [[视频生成]] | 使用 `input.media` 传入 `reference_image` 或 `first_frame`/`last_frame`。注意校验模型对首尾帧分辨率的兼容性，避免画幅裁剪导致画面崩坏。 |
| **3D 资产生产管线、轻量化 WebGL 展示** | [[3D生成]] | 根据渲染精度需求选择模型：端侧/移动端展示选 `Tripo-P1.0`（面数低、加载快）；高精度打样选 `Tripo-H3.1`。**务必在任务成功 2 小时内将 `.glb` 下载并转存至企业 OSS**，避免链接过期导致资产丢失。 |
| **多视角控制与几何精度要求高** | [[3D生成]] | 多图输入模式需严格遵循前/左/后/右数组结构，缺失视角不可省略。若需控制 PBR 材质开关，需在 `parameters` 中显式设置 `pbr: false` 与 `texture: false` 以获取纯网格模型。 |

## 开发者注意事项
1. **异步轮询最佳实践**：视频与 3D 生成均依赖异步任务机制。生产环境强烈建议使用官方 `DashScope SDK`（Python/Java）封装轮询与指数退避重试逻辑，避免高频 HTTP `GET` 请求触发 `task_id` 查询接口的限流（默认 20 RPS）。
2. **地域与鉴权隔离**：三大模块的 `API Key`、计费池与网关 Endpoint 均按地域物理隔离。跨地域混用将直接返回 `InvalidApiKey` 或 `404`。多活部署时需按地域独立配置 Key 池与路由策略。
3. **协议演进与历史路由**：平台正持续向多模态统一路由收敛。视频与 3D 生成当前共用 `/video-generation/` 历史路径分组，请勿在业务网关中误路由；图像生成新业务务必迁移至 `/multimodal-generation/generation`，废弃旧版独立路由（如 `/text2image/image-synthesis`）。
4. **资源持久化与成本优化**：所有多媒体输出 URL 均为临时签名链接。图像生成通常提供数天有效期，视频约 24~72 小时，3D 模型严格限制为 2 小时。架构设计时应内置**结果拉取与云存储归档管道**，避免重复生成导致计费飙升。
