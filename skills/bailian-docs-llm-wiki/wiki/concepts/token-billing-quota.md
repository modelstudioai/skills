# Token 计量与计费

Token 计量与计费是百炼平台衡量 AI 模型调用资源消耗与结算费用的核心机制。平台以输入、输出及上下文缓存 Token 为基本计价单位，结合阶梯定价、额度抵扣链路与多维度监控，为开发者提供透明、可控的成本管理体系。

## 不同场景下的使用方式
* **实时与批量推理**：文本、视觉、语音等多模态模型均按单次请求的实际消耗 Token 量结算。[[批量推理]] 任务享受半价计费优惠，适用于离线数据处理与高吞吐场景。
* **团队订阅（Token Plan）**：面向企业采用 Credits 统一计量。通过专属端点与 `sk-sp-xxx` 格式 API Key 调用，额度按“坐席月度套餐 → 共享用量包 → 暂停服务”优先级抵扣，仅限交互式编程与智能体调用。
* **上下文优化**：启用 [[上下文缓存]] 后，系统自动识别历史 Prompt 片段并应用折扣系数计量。缓存抵扣与常规推理计费互斥，可显著降低长窗口对话成本。
* **训练与部署**：模型微调按“训练 Token 总量 × 循环次数 × 单价”独立核算；部署阶段转为按预置吞吐（TPM）、模型单元（MU）时长或独占实例计费，不纳入实时 Token 按量池。

## 关键参数与配置
* **计价粒度与阶梯阈值**：最小计费单位为 1 Token。部分文本模型按单次输入量划分计费区间（如 0~32K、32K~128K、>128K），命中区间后单次请求全量 Token 均按该档位单价结算。
* **多模态折算公式**：图像/视频理解输入 Token 按 `h × w / (32 × 32) + 2` 动态计算，分辨率提升将非线性增加上下文成本。
* **额度抵扣链路**：系统严格遵循 `免费额度 → 资源包 → 垂直模型节省计划 → AI 通用型节省计划 → 按量付费` 顺序扣减。同类型额度优先抵扣最早到期包，无需手动激活。
* **出账与延迟**：实时推理账单通常在调用后 2～10 分钟生成；批量推理、训练与知识库服务为小时级出账。控制台普通统计面板同步延迟约 1 小时，精确明细需查看账单日志。
* **防超耗拦截**：建议在控制台开启“免费额度用完即停”。额度耗尽时 API 返回 `AllocationQuota.FreeTierOnly`（403）并阻断后续计费请求。

## 监控与成本分账
* **全链路用量追踪**：通过 [[模型监控]] 可查询 RPM、TPM、Token 消耗趋势及首 Token 延时。支持按 `workspace_id`、`model`、`apikey_id` 等标签聚合过滤，定位异常消耗。
* **应用级拆分观测**：[[应用观测]] 自动记录调用链中各节点（LLM、Retriever、Tool）的 Token 输入/输出拆分数据，支持将 Span 数据导出至 [[评测数据集]] 进行成本与效果对齐评估。
* **财务分账与溯源**：下载账单 CSV 后解析 `实例 ID`（含 APIKeyID、业务空间ID、模型名称等）可精准溯源。通过资源管理器为 [[业务空间]] 绑定标签并启用费用标签，可实现 T+1 级项目/部门成本分摊。

## 注意事项
* **额度清零与周期**：套餐包、节省计划及免费额度均按自然月/订阅周期结算，当月未用额度**不累积**，过期自动清零。
* **地域策略差异**：免费额度仅对中国内地版生效；全球、国际及欧盟地域单价因算力调度存在差异，且不提供免费额度。
* **欠费连锁阻断**：账户可用余额 `< 0` 即视为欠费，将全局阻断所有依赖账户余额的服务。建议提前配置预算阈值或消费预警。
* **产品线严格隔离**：[[token-plan]] 与 [[编码计划]] 为完全独立计费体系，Base URL 与 Key 前缀不同。混用将触发 `InvalidApiKey` 鉴权失败或异常扣费，配置时请务必核对端点域名。
* **数据合规边界**：平台采用多租户隔离与全链路 HTTPS 加密，承诺不将业务数据用于基座模型训练。账单与调用日志依法留存，敏感业务建议结合本地加密或私有化部署方案。
