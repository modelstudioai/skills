# data connection overview

数据连接是阿里云百炼平台管理外部数据源的统一入口，支持安全访问企业数据库、文档系统及对象存储。通过构建数据连接器，开发者可在应用对话流程中实现非结构化文档解析、结构化表格检索与数据库实时查询。完整功能清单与操作入口可参考 [数据连接](../../raw/application-user-guide/data-connection-overview/data-connection.md)。

## 支持的连接器类型
按数据存储与访问模式，连接器分为**平台托管**与**流处理**两类：

| 模式 | 类型 | 存储方式 | 核心能力 | 适用场景 |
|:---|:---|:---|:---|:---|
| **平台托管** | 文件 | 平台存储或自有 OSS | 文档切片、向量化、多模态解析 | PDF/Word/Markdown/PPT等非结构化知识管理 |
|  | 表格 | 平台存储或自有 OSS | 表头映射、结构化检索、以图搜图 | CSV/Excel等结构化数据查询与分析 |
| **流处理** | MySQL | 源端保留，实时访问 | SQL 查询执行（仅 DMS 导入方式） | 业务库实时状态查询 |
|  | PostgreSQL | 源端保留，实时访问 | SQL 查询执行、逻辑日志同步 | 复杂关系型数据关联分析 |
|  | PolarDB-X 2.0 | 源端保留，实时访问 | SQL 查询执行、分布式节点访问 | 云原生分布式数据库集成 |
|  | 语雀 | 源端保留，实时访问 | Token 鉴权检索、知识库同步 | 企业文档与 Wiki 内容引用 |
|  | OSS | 源端保留，实时访问 | Bucket 挂载、文件流式读取 | 大规模媒体/日志/备份文件读取 |

## 关键参数与配置
- **基础元数据**：连接器名称与描述。描述文本将作为 Prompt 上下文的一部分，直接影响 [[agent-application]] 的调用意图识别与检索精度。
- **存储位置**：
  - `平台存储`：开箱即用，适合中小规模数据。
  - `自有OSS存储`：需配置目标 Bucket 标签 `bailian-connector-access=ReadAndWrite`，并授予相应 [[ram-permissions]]。
- **网络与连通性**：支持公网/私网。PolarDB-X 仅支持私网；PostgreSQL 需开启 `wal_level=logical` 并配置高权限账号；MySQL 需配置白名单。连通性检测底层依赖 EventBridge 或 [[dts-service]]。
- **数据源接入方式**：
  - `创建自定义数据源`：手动录入连接串、IP/端口、账号密码。
  - `从 DMS 导入数据源`：继承 DMS 已录入实例配置。**仅此方式支持后续直接执行 SQL**。
- **解析策略（文件）**：提供电子文档解析、文档智能解析、大模型文档解析、Qwen VL 解析及音视频解析。需根据文件类型（文本/图像/音视频）选择，图表理解需启用 LLM 或 VL 模式。
- **表格 Schema**：自定义表头时，`列名`、`类型` 为必填。`描述` 字段建议提供自然语言释义以辅助模型理解。若类型设为 `image_url`，必须为公网可访问链接，系统将自动提取图像向量。

## 使用方式
1. **创建连接器**：在控制台单击创建，选择对应类型并填写连接参数。通过“开始检测”验证网络与鉴权后确认提交。
2. **导入数据**：
   - **文件**：进入详情 -> 新建类目 -> 本地上传 -> 配置解析策略与业务标签 -> 提交解析任务。
   - **表格**：新建数据表 -> 选择直接上传 Excel 或自定义列结构 -> 上传数据文件。需严格保证上传文件列数/列名与表定义一致。
3. **应用集成**：在 [[agent-application]] 控制台调试或调用 [[api-calling-guide]] 时，可通过 `tags` 参数过滤已标记文件，提升 [[vector-retrieval]] 命中率。流处理连接器绑定后即可在 Agent 工具调用中透传 SQL。
4. 详细控制台操作路径与界面指引见 [数据连接](../../raw/application-user-guide/data-connection-overview/data-connection.md)。

## 限制与注意事项
- **容量与查看周期**：平台托管提供最多 100,000 个文件 / 1TB 额度。导入后生成独立副本，文件管理页仅展示最近 **90 天**内的导入记录，超期不可见但不会删除。
- **SQL 执行约束**：MySQL、PostgreSQL、PolarDB-X 2.0 仅支持通过 DMS 导入方式创建的实例执行 SQL。自定义录入的实例仅用于元数据注册，无法直接执行查询。
- **PostgreSQL 环境要求**：除 `wal_level=logical` 外，自建实例需修改 `pg_hba.conf` 放行 `100.64.0.0/16` 网段，否则 DTS 检测将失败。
- **OSS 兼容性**：
  - 必须开通向量检索服务。
  - **不支持**归档、冷归档或深度冷归档存储类型的 Bucket。
  - 开启 Referer 防盗链的私有 Bucket，需将 `*.console.aliyun.com` 加入白名单 Referer 列表。
- **格式与结构刚性限制**：文件导入暂不支持 JSON/CSV/YAML，需预先转为 XLSX/XLS。表格结构（列名/类型）创建后**不可修改**，后续导入文件必须完全对齐。

> **注意**：原文档对平台托管存储的说明存在差异：文件存储描述为“限时免费”，而表格存储描述为“额度用完后自动转为按量付费”。计费策略可能随版本迭代调整，实际生产环境部署前请以控制台最新计费规则或 [数据连接](../../raw/application-user-guide/data-connection-overview/data-connection.md) 为准。
