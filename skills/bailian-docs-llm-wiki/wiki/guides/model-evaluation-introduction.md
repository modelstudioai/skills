# model evaluation introduction

模型评测是百炼平台提供的量化验证工具，通过结构化数据集与多维度评分规则，客观评估大语言模型在特定业务或通用场景下的实际表现。该功能支持自定义业务指标与行业基线对比，为模型选型、[[model-fine-tuning]] 效果验证及线上能力监控提供可量化的数据支撑。完整功能架构与场景设计详见 [模型评测](../../raw/model-user-guide/model-evaluation-introduction/model-evaluation-overview.md)。

## 支持的模型/功能
- **模型支持范围**：覆盖千问系列（Max/Plus/Turbo/Coder/Math等）、通义法睿及主流第三方文本生成模型。支持对预置模型进行调优后的专属版本参与评测。具体可用模型以控制台创建任务页实时列表为准。
- **双轨评测模式**：
  - **自定义评测**：面向垂直业务场景，需手动构建评测维度。内置5类评分机制：大模型评估（数值型/分类型）、规则评估（文本相似度/字符串精确匹配）及人工评估。
  - **基线评测**：面向通用能力验证，内置 C-Eval、MMLU、GSM8K、HellaSwag 等行业标准数据集。自动输出分学科得分明细、能力雷达图及业界对标数据。模式差异与选型建议可参考 [模型评测](../../raw/model-user-guide/model-evaluation-introduction/model-evaluation-overview.md)。
- **排行榜能力**：支持将多轮评测任务映射至同一维度，实现跨模型横向对比与可视化排序。

## 关键参数
- **数据源类型**：
  - `评测数据集`：仅含 `Prompt` 与 `Completion`。流程为触发实时推理生成 `Output` 后打分，适用于冷启动或多模型盲测。
  - `推理结果集`：包含 `Prompt`、`Completion` 与 `Output`。直接读取已有输出打分，跳过推理环节，适用于成本控制与基准对照。
- **Prompt 体系**：
  - `System Prompt`：作用于被评测模型，用于设定角色定位或行为规范（任务创建时配置，非必填）。
  - `评分器 Prompt`：作用于裁判模型，定义打分逻辑与分级标准。仅大模型评估维度强制配置，支持预置模板或自定义编写。编写时必须正确引用 `${[[prompt|prompt]]}`、`${output}`、`${completion}` 核心变量。
- **评分阈值**：数值型评分默认量程 0~5 分，需明确定义通过阈值（如 `≥3.0` 计为 Pass），该参数直接决定“通过率”指标的计算逻辑。
- **数据规模建议**：配置验证建议 50~100 条；正式评估建议 200~500 条；全量边界压测建议 500+ 条。

## 使用方式
1. **数据准备**：在 [[data-management]] 模块新建数据集并上传 Excel 文件。严格确保数据类型选择“评测集”，格式列名需与系统要求对齐。
2. **维度配置**：创建评测维度，指定评估类型、裁判模型及 Prompt 逻辑。若采用自定义规则，需在评分器 Prompt 中清晰定义各分数档的判定边界。
3. **任务构建**：关联数据源、目标模型与已创建的维度。基线评测直接勾选预置榜单发起；自定义评测需明确绑定对应数据集与维度。完整交互路径见 [模型评测](../../raw/model-user-guide/model-evaluation-introduction/model-evaluation-overview.md)。
4. **结果解析**：任务状态变更为“评测完成”后，可在“数据明细”逐条核对评分与模型输出，在“指标统计”查看综合得分与通过率。基线评测结果自动附带行业分位对比，便于快速定位能力短板。

## 限制和注意事项
> **注意**：服务仅支持中国内地部署模式。所有接入点与数据存储强制位于**北京地域**，推理计算资源不具备跨地域调度能力。
- 数据隔离要求：评测链路严格校验数据集类型，[[data-preparation]] 过程中产生的训练集或验证集无法直接消费，仅“评测集”类型可被任务调用。
> **注意**：基线评测**仅支持调优后的模型**，不支持直接调用千问-Max 等官方预置基座。若任务配置页模型列表为空或置灰，需优先完成 [[model-fine-tuning]] 与 [[model-deployment]]。
- 计费结构：评测过程产生两路独立费用。① 被评测模型的 Token 推理费用（仅使用“评测数据集”时触发）；② 裁判模型的 Prompt/Completion 调用费用。
- 状态流转：高峰时段任务可能进入“待执行”排队状态，请勿重复提交触发并发冲突。若状态变更为“评测失败”，通常源于 Excel 格式校验拦截、裁判模型服务超时或维度配置逻辑错误，建议核对日志后重试。
