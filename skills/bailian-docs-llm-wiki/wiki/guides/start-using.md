# start using

阿里云百炼平台提供零代码控制台与标准化 API 双轨接入模式，支持开发者快速构建基于私有知识库的智能问答、复杂工作流及自动化智能体应用。平台已深度集成主流大语言模型、多模态解析能力及企业级运维特性，并实现知识库与模型调用的统一资源管理。

## 支持的模型与功能
- **大语言模型**：默认推荐 `qwen-max` 作为主力推理基座；已全面兼容 `QwQ` 深度推理系列、`DeepSeek` 系列及 `Qwen2.5-VL` 多模态视觉模型。
- **知识库能力**：支持标准版文档、数据表、图片/音视频非结构化知识库，内置 `text-embedding-v4` 向量化引擎与图文检索解析能力。
- **应用架构**：`Agent 2.0` 支持将 MCP 服务、插件与知识库统一为自主规划工具；工作流应用新增多模态生成节点、异步运行模式及 Dify 一键导入。完整应用类型对比与快速上手路径请参考 [0代码构建私有知识问答应用](../../raw/application-user-guide/start-using/build-knowledge-base-qa-assistant-without-coding.md)。

## 关键参数
- **System Prompt**：严格定义智能体角色边界、任务逻辑与输出格式规范。
- **检索 TopK**：可独立调整 `初步向量检索TopK` 与 `初步关键词检索TopK` 数值，直接控制送入重排序模型的 Token 量以优化成本。
- **调用控制**：Responses API 调用工作流或智能体时，可通过请求体设置 `background: true` 触发异步任务模式（返回 Task ID 轮询）。
- **权重与策略**：多知识库并发绑定时支持设置检索权重（Weight）；知识库节点提供必定调用/智能调用/兼容旧版三种执行策略。

## 使用方式
- **控制台零代码构建**：访问应用中心创建空白应用 -> 配置模型与提示词 -> 通过[[数据连接器]]上传文档 -> 创建[[知识库]]（推荐智能切分） -> 在应用[[技能]]面板挂载知识库 -> 测试并发布。
- **API 标准化集成**：全面采用 [[Responses API]] 进行统一调用。同步模式适用于实时对话，异步模式适用于长耗时批处理。新版长期记忆、知识库创建与监控均提供独立 OpenAPI，支持外部业务系统直连。详细接口演进历史与参数映射规则可查阅 [应用功能动态](../../raw/application-user-guide/start-using/application-release-notes.md)。
- **高代码扩展**：支持基于 Python 项目结构部署 AI 后端服务，或结合 MCP SDK 将第三方能力集成至个人项目。

## 限制和注意事项
- **商业化计费**：知识库服务自 2026 年 1 月 4 日起正式计费（规格费 + 模型调用费），提供后付费与预付费资源包。上线前请前往控制台核对免费额度余量。
- **接口废弃与迁移**：`[[Assistant API]]` 已进入下线流程，新项目严禁使用，需迁移至 Responses API 或工作流调用链路。`[[长期记忆]]` 已全面升级至 2.0 架构，旧版 API 停止维护，新接口支持跨应用共享与对话自动提取。
- **性能与兼容性约束**：文档解析与向量化通常耗时 1~6 分钟，需合理处理前端加载状态。`QwQ` 模型在特定快照版本中暂不支持插件调用与音视频实时交互能力。
> **注意**：控制台默认配置可能随底层架构升级而调整，若需针对复杂版面（如嵌套表格、多图表混排）提升召回精度，请关闭“智能切分”改用自定义 Chunk 规则，或为文档附加 metadata 过滤条件。更多计费细则与版本兼容性说明，请以官方最新公告 [应用功能动态](../../raw/application-user-guide/start-using/application-release-notes.md) 为准。
