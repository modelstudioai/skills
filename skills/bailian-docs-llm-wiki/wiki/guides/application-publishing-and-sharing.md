# application publishing and sharing

阿里云百炼平台提供标准化的应用发布与分享机制，支持将已构建的 AI 应用快速部署至多类业务终端。开发者可通过官方网页、自定义低代码界面、第三方社交/办公平台及模块化组件等多种渠道交付能力，并支持实时音视频互动。本文档汇总发布渠道、核心参数、集成步骤及架构限制，供开发与集成参考。

## 支持的模型/功能
平台支持将 [[智能体应用]] 与 [[工作流应用]] 发布为以下类型的服务：
- **网页与 UI 终端**：官方临时网页（默认 7 天有效）与自定义 [[UI 应用]]。后者基于拖拽式低代码引擎构建，支持绑定数据库、外部 HTTP 服务及权限路由，可发布至开发环境（免费/24小时有效）或生产环境（长期有效/支持自定义域名）。详见 [UI设计器](../../raw/application-user-guide/application-publishing-and-sharing/ui-designer.md)。
- **第三方生态集成**：钉钉机器人（支持流式消息卡片与群聊交互）、微信公众号（AppID 授权与扫码接入）、音视频实时互动（语音/视频双向交互，提供 H5 扫码体验与 SDK 集成能力）。
- **组件化复用**：将完整应用封装为标准化组件，供其他智能体或工作流跨应用调用，实现能力沉淀与编排加速。

## 关键参数
发布与集成过程中需重点配置以下参数：
- **鉴权与凭证**：[[API Key]]（需与目标应用同属一个业务空间）、钉钉 Client ID/Secret/卡片模板 ID、微信 AppID。
- **组件入参配置**：
  - `query`（系统参数）：传递用户自然语言输入。
  - `imageList`（系统参数）：传递公网图片地址列表，仅对视觉理解模型生效。
  - **别名（Alias）**：对外暴露的调用键名，用于隔离内部参数命名冲突。
  - **可见性**：控制参数是否透传至调用方（如文本模型需隐藏图像参数）。
- **传参方式**：
  - `业务透传`：由调用方手动填充（智能体）或上游节点显式赋值（工作流）。
  - `模型识别`：智能体中由大模型根据参数描述自动从上下文推断填充；工作流中不生效，必须降级为显式传值。
详细组件参数定义与透传逻辑请参考 [使用智能体或工作流作为组件](../../raw/application-user-guide/application-publishing-and-sharing/use-agent-or-workflow-as-component.md)。

## 使用方式
### 1. 基础发布流程
在 [[应用管理]] 控制台点击目标应用的 **发布** 按钮，根据业务场景选择发布渠道。首次接入钉钉/微信需完成计算巢 AppFlow 授权（自动创建 SLR 角色并启用 [[API Key]] 加密传输）。

### 2. 第三方平台接入
- **钉钉**：在钉钉开放平台创建应用 -> 申请 `Card.Streaming.Write` 与 `Card.Instance.Write` 权限 -> 配置消息卡片模板 -> 机器人配置选择 **HTTP 模式** -> 填入百炼生成的回调地址。
- **微信**：绑定公众号开发者 ID，授权后生成专属二维码或 Webhook 接入点。
- **UI 应用**：选择模板 -> 配置数据库表映射（若表不存在则自动创建，结构需严格匹配）-> 拖拽布局 -> 发布并分配访客链接。

### 3. 组件调用集成
- **智能体集成**：在技能列表勾选已发布组件。大模型将基于 `组件描述` 自动决策调用时机。含业务透传参数时，可通过控制台测试页手动赋值，或在 API 请求中传入 `biz_param`。
- **工作流集成**：画布中拖入“组件节点” -> 映射上游变量（如 `系统变量/query`） -> 开启[[streaming-output|流式输出]] -> 将输出对接至后续节点或结束节点。
跨渠道分享机制与权限控制策略详见 [分享智能体应用](../../raw/application-user-guide/application-publishing-and-sharing/share-an-application.md)。

## 限制和注意事项
- **调用链路限制**：严禁循环嵌套（如 A 调 B，B 调 A），否则将导致重复调用死锁。建议控制调用深度（A→B→C），多级串联易触发平台最长运行时间阈值引发超时。
- **工作流文件上传适配**：若工作流定义了 `file` 类型自定义参数，在 UI 设计器发布时必须显式映射变量 `{{{实际字段名:files[0]}}}`，否则前端无法正确读取上传文件。
- **费用归属**：所有通过分享链接或 API 调用产生的 Token 消耗，均计入应用创建者 UID 账号账单。
- **钉钉模式限制**：钉钉机器人消息接收仅支持 HTTP 模式，选择 Stream 模式会导致回调中断。
> **注意**：关于工作流中 `模型识别` 参数的实际表现，早期文档描述曾暗示平台可自动推断，但当前版本中工作流引擎**不支持**自动上下文推断，必须严格按照 `业务透传` 模式由上游节点显式绑定变量值。请以最新控制台行为为准，避免编排失败。
