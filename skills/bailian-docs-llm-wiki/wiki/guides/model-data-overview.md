# model data overview

百炼平台提供完整的模型数据管理功能，涵盖数据集上传、格式校验、版本管理以及可视化的数据清洗与增强流水线。开发者可通过标准化的数据准备流程，高效构建高质量语料以用于[[模型调优]]或[[模型评测]]。本文档梳理了平台支持的数据类型、核心参数规范、处理流程及开发限制。

## 支持的模型与功能

平台根据任务类型将数据集分为**训练集**与**评测集**两大类，具体支持能力如下：

* **训练集**：支持文本生成、多模态理解、图生视频（首帧/首尾帧）。涵盖 SFT（标准对话/深度思考/视觉理解）、DPO（偏好对齐）及 CPT（连续预训练）格式。
* **评测集**：目前仅支持文本生成类评测，采用单轮 Prompt-Completion 结构，用于评估模型泛化能力。
* **数据处理**：提供基于画布的[[数据处理流]]，内置数据清洗（如特殊字符移除、敏感信息打码、相似度去重）与数据增强（基于 Few-Shot 策略扩展样本规模与多样性）。完整数据集分类与管理说明请参考 [训练集与评测集](../../raw/model-user-guide/model-data-overview/training-set-and-evaluation-set.md)。

## 关键参数与格式规范

### 通用对话与对齐（SFT / DPO / 视觉）
* 采用 ChatML 格式，每行为独立 JSON 对象。不支持 OpenAI 的 `name`、`weight` 参数，所有 `assistant` 输出默认参与训练。
* `loss_weight`（0.0~1.0）：用于调节单条数据或单行回复的训练重要性。**该参数目前为邀测功能**，需联系商务开通。
* **思考模式（Thinking）**：仅最后一个 `assistant` 回复可包含 `<think>\n思考内容\n</think>\n\n` 标签。中间轮次禁止添加思考标签；若样本中不包裹该标签，训练后不建议开启思考模式调用。

### 视觉理解与图生视频
* **视觉模型 System 提示**：必须使用数组格式 `[{"text": "内容"}]`，禁止直接使用字符串。
* **视频输入模式**：支持路径模式（`video: "file.mp4"`）与帧列表模式（`video: ["0.jpg", "1.jpg"]`，仅 qwen3.5+ 支持）。可配置 `fps`、`video_start`、`video_end`、`resized_width/height`。
* **包结构与限制**：
  * 训练文本必须固定命名为 `data.jsonl` 并置于 ZIP 根目录。
  * 视觉训练包最大支持 2GB；图生视频 `data.jsonl` 单文件上限 20MB。
  * 图片单张 ≤10MB 且宽高 ≤1024px；视频最大分辨率 4096×4096，支持 BMP/JPEG/PNG/WEBP/MP4/MOV。文件名仅支持 ASCII 字母、数字、下划线、连字符。

### CPT 预训练
* 格式极简，单行为 `{"text":"纯文本内容"}`。建议基线规模 ≥1000 万 Token 优质数据。

## 使用方式

1. **数据准备与上传**：按对应格式生成 JSONL/Excel/ZIP 文件，上传至业务空间的[[模型数据]]模块。系统支持版本管理，新旧数据隔离存储。
2. **数据清洗与增强**：通过控制台创建[[数据处理流]]（开始节点 → 数据清洗节点 → 数据增强节点 → 结束节点）。处理任务执行后会自动生成新数据版本，原始数据不受影响。详细流水线搭建指南请参考 [数据清洗或增强](../../raw/model-user-guide/model-data-overview/data-processing.md)。
3. **任务投递**：数据就绪后，在[[模型调优]]或[[模型评测]]控制台选择对应版本提交训练或评估任务。
4. **规模建议基线**：SFT ≥1000 条，DPO ≥100 条，CPT ≥1000 万 Token。若效果不佳，优先补充高质量样本或引入[[知识库]]增强，而非盲目扩大规模。

## 限制和注意事项

> **注意**：
> * **地域限制**：本文所述模型数据功能仅适用于**中国大陆版（北京地域）**，其他地域暂不支持。
> * **API 支持状态**：数据清洗与增强功能目前**仅提供控制台可视化操作**，暂未开放 API 接口。
> * **处理集范围限制**：[[数据处理流]]当前仅支持 SFT-文本生成训练集，**不支持**视觉理解（SFT-VL）与 DPO 格式。法律、医学、文学等强专业/低结构化文本建议跳过自动处理环节，直接用于微调或检索增强。
> * **格式校验冲突提示**：早期部分示例文档中视觉模型的 `system` 字段允许使用字符串，但当前平台校验已强制要求数组结构 `[{"text":"..."}]`。请以控制台最新校验报错为准。`loss_weight` 等高级参数若未开通白名单，上传或训练时将直接忽略。
